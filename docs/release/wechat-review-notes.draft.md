## 二、核心功能说明

本小程序的核心功能是一个基于规则的菜品推荐工具，不含 AI 服务、地图定位、在线支付。

### 功能清单

| 功能 | 说明 | 是否需要登录 |
|------|------|:--:|
| 一键推荐 | 根据筛选条件推荐一道菜品，展示名称、分类、标签、参考价位和推荐理由 | 否 |
| 条件筛选 | 按餐段、价位、口味、食物类型、菜系进行组合筛选 | 否 |
| 换一个 | 在已推荐基础上换一道菜，排除已看过的结果 | 否 |
| 我就吃它 | 决定吃当前推荐的菜品，记录到饮食历史 | 是（未登录会引导登录） |
| 吃完评分 | 用餐完成后对菜品打分（1-5 星）和添加备注 | 是 |
| 历史记录 | 查看过往的饮食决定和评分记录 | 是 |
| 黑名单 | 将不想再看到的菜品加入黑名单，后续推荐自动排除 | 是 |
| 暂时不想吃 | 按菜品分类设置"暂时不想吃"（1-30 天），到期自动恢复 | 是 |
| 意见反馈 | 提交功能建议、问题反馈、推荐不准等反馈 | 否（支持匿名） |

### 关于 AI 的说明

**本小程序不接入任何 AI 服务。** 推荐算法基于规则打分（黑名单过滤 → 不想吃过滤 → 最近吃过扣分 → 餐段/价位/口味匹配加分 → 从 Top 5 随机选一个），不含大语言模型、机器学习模型或任何 AI/LLM 相关依赖。

## 三、审核测试路径

审核员可按照以下步骤体验完整功能流程：

### 路径 A：匿名推荐体验

1. 打开小程序 → 进入首页
2. 首页顶部展示筛选条件栏（餐段/价位/口味/分类）
3. 点击"帮我选一个"按钮 → 看到推荐结果卡片（菜品名、分类标签、参考价位、推荐理由）
4. 点击"换一个" → 看到另一道推荐菜品
5. 切换筛选条件（如更换餐段为"午餐"、价位选"15-25"）→ 再次点击"帮我选一个" → 结果随条件变化

### 路径 B：登录 + 完整记录流程

1. 在首页点击推荐结果卡片下方的"我就吃它"
2. 未登录用户 → 跳转到登录页
3. 登录页显示"今天吃啥"品牌视觉 + "微信登录"按钮
4. 点击"微信登录" → 调用微信授权
5. 登录成功后自动跳回首页，并记录用餐决定
6. 首页卡片区变为"当前已决定"状态 → 显示菜品名、餐段、决定时间
7. 点击"吃完去评分" → 进入记录页
8. 记录页显示菜品信息 → 可选择评分（1-5 星）→ 可填写备注 → 点击"提交"
9. 提交成功后自动返回首页

### 路径 C：个人中心

1. 从首页品牌头像进入"我的"页面
2. 查看头像、昵称、登录状态
3. 依次点击"吃过记录""黑名单""暂时不想吃"
4. "吃过记录"页展示历史饮食记录列表 → 可点击查看详情和评分
5. "黑名单"页展示已拉黑的菜品列表 → 可删除（右滑或按钮）
6. "暂时不想吃"页展示当前生效的分类过滤 → 可删除
7. "意见反馈" → 进入反馈页 → 选择类型、填写内容 → 提交成功

### 路径 D：意见反馈

1. "我的" → "意见反馈"
2. 选择反馈类型（功能建议/问题反馈/推荐不准/界面体验/其他）
3. 可选填满意度评分
4. 填写反馈内容（5-500 字）
5. 可选填联系方式
6. 点击"提交" → 提示"已收到反馈" → 自动返回

## 五、隐私信息收集说明

以下是在微信公众平台"隐私保护指引"中需要声明的信息类型，以及对应的收集目的和场景：

| 信息类型 | 收集目的 | 收集场景 |
|---------|---------|---------|
| 微信 OpenID（用户标识） | 账号登录、识别用户身份、关联饮食记录和偏好 | 用户点击"微信登录"时 |
| 用户昵称/头像 | 在"我的"页面展示个人形象 | 登录时（如用户提供） |
| 饮食记录（菜品、餐段、时间、评分、备注） | 展示历史记录、优化推荐 | 用户点击"我就吃它""吃完评分"时 |
| 黑名单/不想吃偏好 | 过滤推荐结果 | 用户在黑名单/不想吃页面操作时 |
| 反馈内容与联系方式 | 处理用户反馈和改进产品 | 用户提交意见反馈时 |
| 设备信息（型号、系统、微信版本） | 定位兼容性问题、排查反馈场景 | 仅在提交意见反馈时自动附带 |

**注意：**
- 具体选项名称以微信公众平台后台"用户隐私保护指引"中的可选项为准，不要硬编后台不存在的选项。
- 筛选偏好（餐段、价位、口味等）仅存储在设备本地，不上传至服务器。

## 六、不涉及能力声明

当前版本**明确不涉及**以下能力，无需在审核材料中勾选或声明：

| 不涉及的能力 | 说明 |
|------------|------|
| 地理位置/定位 | 无地图、无附近推荐、无 POI，不调用 `wx.getLocation` |
| 附近餐厅/商家信息 | 核心定位是推荐"吃什么"，不是"去哪吃" |
| 外卖下单 | 无外卖平台对接、无跳转点餐链接 |
| 在线支付 | 无支付功能，不调用 `wx.requestPayment` |
| 手机号获取 | 登录页明确标注"仅获取基本信息，不会获取你的手机号"，不调用 `wx.getPhoneNumber` |
| 通讯录 | 不调用通讯录相关接口 |
| 相册/摄像头 | 不调用 `wx.chooseImage`、`wx.chooseVideo` 等 |
| 录音/麦克风 | 不调用录音相关接口 |
| 广告追踪 | 无广告 SDK、无 IDFA/IMEI 采集、无用户画像用于广告投放 |
| AI/大模型服务 | 不使用 OpenAI、LangChain 或任何 AI 相关 SDK |

## 七、微信公众平台隐私保护指引填写建议

在微信公众平台后台 → 设置 → 服务内容声明 → 用户隐私保护指引中，根据当前功能建议声明以下信息类型：

1. **用户标识信息**（如 OpenID）
   - 用于账号登录和区分用户
   - 通过微信登录接口获取

2. **用户昵称/头像**（如代码实际使用）
   - 用于个人中心展示
   - 在个人中心"我的"页面展示

3. **用户操作记录 / 服务记录**
   - 用于饮食记录、评分、黑名单、反馈归属
   - 用户通过"我就吃它""吃完评分"等主动操作产生

4. **设备信息**
   - 用于反馈问题排查和兼容性定位
   - 仅在用户提交"意见反馈"时附带

5. **用户反馈内容**
   - 用于处理用户问题和改进产品

**重要提示：**
- 以上信息类型名称以微信公众平台后台实际可选项为准。
- 如果后台要求的选项与上述不完全匹配，请选择描述最接近的选项。
- 不要勾选任何实际未收集的信息类型（如位置信息、手机号、相册等），否则可能导致审核不通过。

## 八、上线前依赖

以下事项必须在提交审核前完成：

### 必须完成

| 项 | 说明 |
|----|------|
| ICP 备案 | 生产域名必须完成 ICP 备案，备案周期约 15-20 个工作日 |
| HTTPS 域名可用 | `api.your-domain.com` 已配置 SSL 证书且可访问 |
| request 合法域名配置 | 在微信公众平台后台 → 开发管理 → 开发设置 → 服务器域名中配置 `https://api.your-domain.com` |
| `WECHAT_MOCK_ENABLED=false` | 生产环境关闭微信 Mock 登录 |
| 生产 AppID/AppSecret | 使用正式小程序的 AppID 和 AppSecret，不使用测试号 |
| 小程序前端 baseUrl | `miniprogram/config/index.ts` 中的 `baseUrl` 指向生产 HTTPS 域名 |
| 隐私保护指引已填写 | 微信公众平台后台已设置用户隐私保护指引 |

### 建议完成

| 项 | 说明 |
|----|------|
| 后端服务健康检查 | `GET /api/health` 返回 `{"status":"UP"}` |
| 数据库初始化 | Flyway 迁移全部执行成功 (`SELECT * FROM flyway_schema_history`) |
| 菜品数据验证 | `SELECT COUNT(*) FROM foods` 预期返回 73 条数据 |
| Nginx HTTPS 配置 | 80/443 端口正常，SSL 证书有效期充足 |
| JWT_SECRET / ADMIN_TOKEN | 已生成强随机密钥，不使用默认值 |
