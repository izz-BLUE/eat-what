# 今天吃啥 - 推荐算法设计

## 概述

本项目采用**纯规则打分**推荐算法，不接入任何 AI 服务。

算法目标：
- 根据用户偏好推荐食物
- 降低最近吃过食物的推荐概率
- 避免推荐黑名单的食物
- 避免推荐有效期内不想吃的食物分类
- 从 Top 5 中随机选一个，避免结果固定

## 算法流程

```
┌─────────────────────────────────────────────────────────────┐
│                    推荐算法主流程                            │
├─────────────────────────────────────────────────────────────┤
│  1. 获取候选食物池（enabled=1）                              │
│     ↓                                                        │
│  2. 排除指定菜品（excludeFoodIds）                           │
│     ↓                                                        │
│  3. 排除用户黑名单（user_blacklist）                         │
│     ↓                                                        │
│  4. 排除有效期内不想吃分类（user_dislikes）                  │
│     ↓                                                        │
│  5. 排除用户选择的分类（categories，最多3个）                │
│     ↓                                                        │
│  6. 口味硬过滤（按逗号拆分精确匹配）                        │

│ 10. 返回推荐结果                                             │

```

## 口味硬过滤规则

| 用户选择 | 过滤规则 |
|----------|----------|
| 不限/空 | 不限制 |
| 清淡 | 必须包含"清淡"，同时不能包含辣、微辣、麻 |
| 不辣 | 排除辣、微辣、麻 |
| 辣 | 必须包含辣或微辣 |
| 重口 | 包含辣、麻、酸之一，或者同时包含咸和香 |

## 详细设计

### 1. 获取候选食物池

从数据库获取所有启用的食物：

```sql
SELECT id, name, category, taste_tags, price_level
FROM foods
WHERE enabled = 1
```

---

### 2. 过滤排除食物

#### 2.1 排除指定菜品（已实现）

排除前端传入的 excludeFoodIds，用于"换一个"功能。

#### 2.2 黑名单过滤（已实现）

```sql
SELECT food_id
FROM user_blacklist
WHERE user_id = ?
```

**处理方式**：直接从候选池中移除，不参与后续打分。

#### 2.3 不想吃分类过滤（已实现）

```sql
SELECT category
FROM user_dislikes
WHERE user_id = ? AND expires_at > NOW()
```

**处理方式**：只查询未过期记录，将匹配分类的食物直接从候选池中移除。过期记录不参与过滤。

---

### 3. 计算每个食物得分

总分公式（当前已实现）：

```
总分 = 餐段匹配 + 价格匹配 + 口味匹配 + 最近吃过扣分 + 随机因素
```

#### 3.1 餐段匹配加分（+30）

根据用户选择的餐段推荐合适的食物类型。

| 餐段 | 匹配的食物分类 |
|------|----------------|
| 早餐 | 快餐、小吃、面食 |
| 午餐 | 快餐、小吃、面食、川菜、粤菜、湘菜、家常菜 |
| 晚餐 | 火锅、烧烤、川菜、粤菜、西餐、日料、韩餐 |
| 夜宵 | 小吃、烧烤、快餐 |

匹配则 +30，不匹配则 +0。

#### 3.2 价格匹配加分（+20）

根据用户选择的价格偏好匹配食物价格等级。

| 价格偏好 | 匹配的 priceLevel |
|----------|-------------------|
| 15以内 | priceLevel = 1 |
| 15-25 | priceLevel = 2 |
| 25-40 | priceLevel = 3 |
| 不限 | 不加分 |

匹配则 +20，不匹配则 +0。

#### 3.3 口味匹配加分（+20）

根据用户选择的口味偏好匹配食物口味标签。

| 口味偏好 | 匹配的 tasteTags |
|----------|------------------|
| 清淡 | 包含"清淡"或"鲜" |
| 重口 | 包含"咸"或"香" |
| 辣 | 包含"辣"或"麻" |
| 不辣 | 不包含"辣"和"麻" |

匹配则 +20，不匹配则 +0。

#### 3.4 最近吃过扣分（已实现）

**数据来源**：`eat_records` 表，使用 `MAX(eaten_at)` 获取最近一次。

**扣分规则**（使用 Duration 精确比较）：

| 距今时间 | 扣分 |
|----------|------|
| < 24 小时 | -100 |
| 24h ~ < 48h | -80 |
| 48h ~ < 72h | -60 |
| 72h ~ <= 168h（7天） | -30 |
| > 168h | 0 |

**实现**：
```java
Duration duration = Duration.between(lastEatenAt, now);
if (duration.compareTo(Duration.ofHours(24)) < 0) return -100;
if (duration.compareTo(Duration.ofHours(48)) < 0) return -80;
if (duration.compareTo(Duration.ofHours(72)) < 0) return -60;
if (duration.compareTo(Duration.ofHours(168)) <= 0) return -30;
return 0;
```

#### 3.5 随机因素（0-19）

加入随机性，避免推荐结果固化。

```java
int randomBonus = new Random().nextInt(20);  // 0-19
```

---

### 4. 从 Top 5 中随机选一个

**目的**：避免每次推荐结果固定，增加惊喜感。

**实现**：
```java
List<Food> top5 = candidates.stream()
    .sorted((a, b) -> Integer.compare(b.score, a.score))
    .limit(5)
    .collect(Collectors.toList());

Food recommended = top5.get(new Random().nextInt(top5.size()));
```

---

### 5. 返回推荐结果

**返回数据**：
```json
{
  "food": {
    "id": 1,
    "name": "猪脚饭",
    "category": "快餐",
    "tasteTags": "咸,香",
    "priceLevel": 2,
    "imageUrl": ""
  },
  "score": 45,
  "reasons": ["适合当前餐段", "符合预算", "最近几天没吃过，换换口味"]
}
```

**推荐理由**：
- 餐段匹配：`"适合当前餐段"`
- 价格匹配：`"符合预算"`
- 口味匹配：`"符合口味偏好"`
- 最近没吃过：`"最近几天没吃过，换换口味"`

## 特殊场景处理

### 场景 1：没有候选食物

**原因**：所有食物都被排除（黑名单、excludeFoodIds、不想吃分类）

**处理**：返回 null，接口返回 `"没有找到合适的菜品"`

### 场景 2：userId 为空

**处理**：不查询黑名单、不查询不想吃记录、不查询最近吃过记录，只做基础推荐

### 场景 3：连续"换一个"超过候选数量

**处理**：当 excludeFoodIds 排除所有候选后，返回 null

#### 黄焖鸡
