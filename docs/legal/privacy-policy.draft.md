"今天吃啥"小程序（以下简称"我们"或"本小程序"）深知个人信息对您的重要性。我们致力于保护您的个人信息安全，并遵循合法、正当、必要、诚信的原则处理您的个人信息。

本隐私政策将帮助您了解我们收集哪些信息、如何使用这些信息以及如何保护这些信息。

## 一、我们收集的信息

### 1.1 微信登录标识

当您使用微信登录时，我们通过微信统一的登录流程获取您的**微信 OpenID**。OpenID 是微信为每个用户在每个小程序中生成的唯一标识符，用于在本小程序内识别您的身份，以便保存您的饮食记录、偏好设置和反馈。

**说明：**
- 此过程由微信官方接口完成，我们不接触、不收集您的微信密码。
- `wx.login()` 返回的临时 code 仅用于服务端换取 OpenID，不存储在您设备或我们服务端以外的任何地方。
- 微信会话密钥（session_key）和 UnionID 不会被我们存储或使用。

### 1.2 用户基础资料

您的**昵称**和**头像**可能会在以下场景中涉及：
- 当后端登录接口收到这些信息时，会保存在您的账户中。
- 当前版本的标准登录流程仅通过 `wx.login()` 获取 OpenID，不会主动调用微信头像/昵称相关接口。
- 如果这些信息被提供，它们仅用于在"我的"页面展示您的个人形象，不会用于其他目的。

### 1.3 饮食记录

当您使用"我就吃它"、"吃完评分"等功能时，我们会记录以下信息：
- 您选择的**菜品**（foodId）
- **餐段**（早餐、午餐、晚餐或夜宵）
- **决定时间**和**用餐完成时间**
- 您主动提供的**评分**（1-5 星，可选）
- 您主动填写的**备注**（最长 256 字，可选）

**用途：** 展示历史记录、优化推荐算法（如"最近吃过扣分"）、生成个性化推荐理由。

### 1.4 饮食偏好

当您使用以下功能时，我们会记录相应的偏好数据：

| 功能 | 记录内容 |
|------|---------|
| 黑名单 | 您不想再看到的菜品 ID，以及拉黑原因（当前固定为"不喜欢"） |
| 暂时不想吃 | 您暂时不想看到的菜品分类（如"川菜""快餐"）和过期时间（1-30 天） |

**用途：** 过滤推荐结果，避免推荐您不想要的菜品。

此外，您在首页设置的**筛选条件**（餐段、价位、口味、食物类型、菜系）仅保存在您设备本地（微信 `wx.setStorageSync`），**不会上传到我们的服务器**。

### 1.5 意见反馈

当您提交意见反馈时，我们会收集：

| 字段 | 是否必填 | 说明 |
|------|:--:|------|
| 反馈类型 | 是 | 功能建议 / 问题反馈 / 推荐不准 / 界面体验 / 其他 |
| 反馈内容 | 是 | 5-500 字 |
| 满意度评分 | 否 | 1-5 星，可选 |
| 联系方式 | 否 | 微信号/手机/邮箱等，由您自行决定是否填写 |

如果您已登录，反馈会关联到您的用户 ID；如果您未登录，反馈将**匿名提交**。

### 1.6 设备与系统信息

在提交意见反馈时，我们会自动采集以下设备信息：

- 设备型号（如 iPhone 15 Pro）
- 操作系统版本（如 iOS 17.4）
- 平台类型（iOS / Android / 开发工具）
- 微信客户端版本
- 微信 SDK 版本

**用途：** 定位兼容性问题、辅助排查反馈场景。**不用于广告追踪。**

**说明：** 设备信息仅在您提交反馈时采集，不会在后台自动采集，不会关联您的浏览行为或其他使用习惯。

### 1.7 日志信息

我们的后端服务在运行过程中可能自动产生以下日志：
- 接口访问日志（不含请求 IP、User-Agent 或请求体内容）
- 异常/错误日志（含错误堆栈信息）
- 微信登录异常日志（含 OpenID，用于排查登录故障）
- 反馈提交记录（含用户 ID 和反馈类型）

**用途：** 安全审计、故障排查、保障服务稳定性。

## 二、我们如何使用信息

我们使用收集的信息用于以下目的：

1. **提供推荐服务：** 根据您的偏好和历史记录生成个性化的菜品推荐。
2. **保存饮食记录：** 记录您的用餐历史，支持查看、评分和备注。
3. **优化推荐算法：** 根据评分和记录调整推荐策略（如"最近吃过扣分""高评分菜品加权"）。
4. **实现黑名单/不想吃功能：** 正确过滤您不想看到的菜品。
5. **处理意见反馈：** 接收、处理和改进您提出的问题或建议。
6. **故障排查与安全保障：** 通过日志信息定位异常、防范恶意请求、保障服务可用性。

## 三、信息存储

### 3.1 服务端存储

您的饮食记录、黑名单、不想吃分类和反馈数据存储在**后端服务器数据库**中。我们采取合理的访问控制和安全措施保护这些数据。

### 3.2 本地存储

小程序可能通过微信提供的 `wx.setStorageSync` 接口在您设备本地存储以下信息：

| 存储内容 | 用途 |
|---------|------|
| 登录状态（token、userId、nickname、avatarUrl） | 保持登录态，下次打开无需重新登录 |
| 筛选偏好（餐段、价位、口味等） | 记住您的筛选条件，下次打开自动恢复 |
| 当前就餐决定（菜品、餐段、决定时间） | 即使退出小程序，回来仍能看到"当前已决定吃什么" |
| 推荐选项缓存（菜品分类、标签等显示用的元数据） | 减少网络请求，加快页面加载速度 |

您可以通过微信的"清除缓存"功能或在小程序内"退出登录""重置筛选"来清除这些本地数据。

### 3.3 存储期限

- 您的饮食记录和偏好数据将在您使用服务期间持续保存。
- 如果您希望删除数据，可通过**意见反馈入口**或发送邮件至 **{联系邮箱}** 联系我们处理。
- 后端日志文件按时间和大小轮转，不会永久保留。

## 四、信息共享

我们**不会**向任何第三方出售、出租或交易您的个人信息。

以下情况下可能涉及信息共享或交互：

1. **微信平台：** 登录功能依赖微信官方 `jscode2session` 接口，您的 OpenID 由微信平台基于其自身的隐私政策处理。
2. **云服务/基础设施：** 我们的后端服务部署在云服务器上，数据库运行在服务器本地，不涉及第三方数据库即服务（DBaaS）或对象存储服务。
3. **法律要求：** 如法律法规或行政、司法机关要求，我们可能在合理必要的范围内披露相关信息。

**如未来接入新的第三方服务**（如云存储、短信服务、统计分析 SDK），我们将更新本隐私政策并通过小程序内公告等方式通知您。
